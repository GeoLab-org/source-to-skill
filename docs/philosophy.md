# Philosophy

The central rule is simple: not every source should become a skill.

A useful agent skill is not a summary. It changes how an agent behaves in a
future task. For that to work, the source needs reusable judgment, evidence,
scope, and a repeatable output shape.

Books often have enough structure for this. A single meeting often does not.
That does not make the meeting worthless. It may become a note, a seed, or a
fold-in update to an existing skill.

`source-to-skill` treats generation as a consequence of readiness, not the
default action.

## What Counts As Reusable Knowledge

Reusable knowledge usually has at least one of these forms:

- a decision rule
- a repeatable workflow
- a checklist
- a named framework
- a failure mode
- a comparison table
- an example that teaches a general move

One-off status updates, local meeting logistics, and loose opinions do not need a
standalone skill.

## RAG And Skills

RAG helps an agent retrieve knowledge.

Skills help an agent act with knowledge.

That difference matters. A skill should tell the agent when to use the source,
what to check, what not to claim, and which evidence supports the guidance.
