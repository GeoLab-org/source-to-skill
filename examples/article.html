Skill Gate Notes

Principle

When a source is short, prefer a seed before a full skill because the evidence base is thin.

Rules

  • Use a mini skill when the source has a focused reusable workflow.
  • Avoid promoting one meeting into a permanent methodology.
  • Prefer fold-in when the source updates an existing skill.

Example

For example, a review note can add one rule to an existing design-review skill without becoming a new skill.
