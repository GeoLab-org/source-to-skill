# Review Playbook for Source-Derived Skills

## Principle

When a source contains reusable judgment, prefer the smallest artifact that can
preserve it without overstating it. A full skill should be reserved for sources
with structure, examples, and repeatable decisions.

## Rules

- Use a full skill when the source has chapters, examples, and repeatable
  decision rules.
- Use a mini skill when the topic is focused but the source is not broad enough
  for a full reference system.
- Use a skill seed when the source has promising rules but lacks enough examples.
- Avoid publishing a full skill from one meeting because one-off context becomes
  noise.
- Prefer folding weak but useful material into an existing skill.

## Example

For example, a design review transcript may contain one durable judgment: AI
visuals should not become modeling instructions until a handoff brief identifies
which parts are spatial logic and which parts are only mood. That rule can become
a seed or fold into an existing design-review skill. The full meeting should not
be promoted into its own skill.

## Checklist

1. Check the first user and task.
2. Check whether the source has repeatable decisions.
3. Check whether examples support the decisions.
4. Check whether the output can guide a future task.
5. Check what the source does not support.
